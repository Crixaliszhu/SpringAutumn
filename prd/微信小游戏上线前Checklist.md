# 《春秋问鼎》微信小游戏上线前 Checklist

> 生成日期：2026-06-27
> 当前阶段：微信小游戏真机已跑完整游戏流程，未发现 UI 显示问题。
> 目标：从“真机可玩”推进到“可灰度、可提审、可回滚”。

---

## 一、上线前总原则

### 1. 发布口径

- [ ] 本次发布定义为 V1.0 微信小游戏首个可公开测试版本。
- [ ] 不再临时加入 V1.1 新功能。
- [ ] 只处理 P0/P1 阻断问题和必要的上线准备。
- [ ] 所有已知问题都有分级、负责人和处理结论。
- [ ] 有明确的灰度版本号、构建时间和 Git 提交记录。

### 2. 问题分级

| 级别 | 定义 | 上线前要求 |
|---|---|---|
| P0 | 启动失败、崩溃、白屏、坏档、核心流程无法完成 | 必须清零 |
| P1 | 严重影响核心体验、误导玩家、数据异常、明显卡顿 | 必须清零或书面豁免 |
| P2 | 体验瑕疵、数值节奏问题、轻微适配问题 | 可带入灰度，但必须记录 |
| P3 | V1.1 功能增强或非关键优化 | 进入后续 backlog |

## 三、微信小游戏构建与包体

### 1. 构建产物

- [ ] 微信小游戏工程可从当前 Unity/Tuanjie 工程稳定导出。
- [ ] 微信开发者工具可打开导出工程。
- [ ] 模拟器可启动到游戏首屏。
- [ ] 真机预览可扫码进入。
- [ ] Release 构建可启动，不依赖 Development Build 日志。
- [ ] 构建流程可复现，关键步骤已记录。

### 2. 包体与资源

- [ ] 首包大小符合当前微信小游戏平台限制。
- [ ] 总包体大小符合当前微信小游戏平台限制。
- [ ] 不必要的测试图片、临时截图、未使用素材未进入发布包。
- [ ] 只打入 V1 必需场景。
- [ ] 字体资源不会导致首包异常膨胀。
- [ ] 配置 JSON 被正确打入可访问路径。
- [ ] 图片、音频、字体资源在真机上均能正常加载。
- [ ] 若包体接近上限，已规划分包或远程资源方案。

### 3. 分包与加载

- [ ] 首次进入只加载必要资源。
- [ ] 后续地图、音效、扩展素材不阻塞首屏。
- [ ] 网络较慢时有可接受的等待体验。
- [ ] 加载失败时不会白屏或卡死。
- [ ] 重新进入小游戏后资源状态正常。

## 四、核心流程回归

### 1. 启动与首局

- [ ] 首次启动无白屏、无崩溃。
- [ ] 新开局能进入天下地图。
- [ ] 顶部年月、粮、钱、人口、兵力显示正常。
- [ ] 暂停、恢复、速度切换正常。
- [ ] 天下地图与区域地图切换正常。

### 2. 经营闭环

- [ ] 玩家村可查看户数、人口、粮、钱、兵、民心。
- [ ] 建设农田/民居/粮仓/训练场/木墙流程正常。
- [ ] 建设队列按月推进并完成。
- [ ] 建筑等级、建筑效果能在 UI 中正确体现。
- [ ] 月度粮食和铜钱结算正常。
- [ ] 资源不会异常变负或溢出。

### 3. 军事闭环

- [ ] 征兵按钮可用。
- [ ] 征兵数量限制符合当前设计。
- [ ] 征兵队列按月推进并完成。
- [ ] 完成征兵后守军正确增加。
- [ ] 可选择目标据点并发起进攻。
- [ ] 可输入或调整出兵数量。
- [ ] 行军、战斗、占领流程正常。
- [ ] 占领后据点归属、守军、民心、地图表现同步更新。
- [ ] 游戏结束判定可触发，且不会重复弹出异常状态。

### 4. 存档读档

- [ ] 保存按钮可用。
- [ ] 读档按钮可用。
- [ ] 至少连续保存/读取 3 次正常。
- [ ] 关闭小游戏后重新进入，存档仍可读取。
- [ ] 读档后顶部 HUD 数据正确。
- [ ] 读档后日志、地图、右侧操作面板状态正确。
- [ ] 占领后的据点归属、建设队列、征兵队列、军队状态不丢失。
- [ ] 存档损坏或不存在时有可接受的提示，不会卡死。

## 七、UI 与体验最终检查

### 1. 布局

- [ ] 顶部 HUD 不被微信胶囊按钮遮挡。
- [ ] 左侧日志区域不会遮挡主要地图操作。
- [ ] 右侧操作区按钮不出界。
- [ ] 菜单弹窗居中。
- [ ] 保存/读取按钮在弹窗内部。
- [ ] 村庄操作菜单位置稳定。
- [ ] 进攻据点列表可读、可点。
- [ ] 出兵数量输入面板可读、可点。

### 2. 文案

- [ ] 所有中文正常显示。
- [ ] 按钮文案清楚，不像未完成占位。
- [ ] 外交、未开放功能若仍存在入口，文案明确“后续版本开放”。
- [ ] 错误提示不会暴露内部路径或技术栈细节。
- [ ] 新玩家能从界面理解下一步可以建设、征兵、进攻。

### 3. 操作

- [ ] 主要按钮点击热区足够大。
- [ ] 地图点击不会穿透 UI。
- [ ] 快速连续点击不会触发异常状态。
- [ ] 弹窗打开后背景操作被正确阻止。
- [ ] 返回、关闭、取消操作都有明确结果。

## 九、合规与提审材料

### 1. 账号与类目

- [ ] 小游戏账号主体信息正确。
- [ ] 服务类目选择正确。
- [ ] 游戏名称与实际内容一致。
- [ ] 游戏简介不夸大、不涉及未实现功能。
- [ ] AppID 使用正确，不混用测试号和正式号。

### 2. 隐私与用户协议

- [ ] 已配置用户隐私保护指引。
- [ ] 已明确收集的数据类型。
- [ ] 已说明数据用途。
- [ ] 不收集与游戏无关的个人信息。
- [ ] 若接入登录、排行榜、广告、支付，隐私说明已覆盖。
- [ ] 游戏内有可访问的隐私协议或平台要求的入口。

### 3. 内容合规

- [ ] 游戏内容不含明显违规政治、色情、赌博、血腥暴力、诱导分享等内容。
- [ ] 春秋历史题材描述避免现实政治影射。
- [ ] 未成年人相关限制按平台要求处理。
- [ ] 若含充值、广告激励、排行榜，规则说明清楚。
- [ ] 不使用未经授权的字体、图片、音乐、商标素材。

### 4. 提审素材

- [ ] 小游戏图标。
- [ ] 小游戏名称。
- [ ] 简介与详细介绍。
- [ ] 至少一组清晰截图。
- [ ] 分享图或宣传图。
- [ ] 隐私协议链接或后台配置。
- [ ] 版本更新说明。
- [ ] 测试账号或测试说明。
